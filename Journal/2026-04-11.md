### Key Decisions Made

**Service Details:**
- Name: Proxly
- Package: com.infiforge.proxly
- Port Range: 8600-8699
- Supports: WireGuard, OpenVPN, IPSec, HTTPS, SOCKS4, SOCKS5

**Backend Libraries (11 from ~/work/libraries/deno):**
1. @infiforge/auth - JWT/OAuth
2. @infiforge/config - Config with all MongoDB auth types
3. @infiforge/database - MongoDB
4. @infiforge/logger - Logging
5. @infiforge/network - HTTP + WS + GraphQL combined (contains hono)
6. @infiforge/redis - Cache/sessions
7. @infiforge/routes - Universal routes
8. @infiforge/security - Encryption/CORS
9. @infiforge/setup - Setup utilities
10. @infiforge/static - Static files
11. rxjs (npm:) - Reactive streams

**Frontend Libraries (from ~/work/libraries/flutter):**
- infiforge_auth, infiforge_network, infiforge_setup, uxage
- flutter_riverpod, rxdart

**Server.ts Pattern:**
- No direct hono import (via network)
- GraphQL via network.graphql
- Each library has .configure(proxly, options) method
- Static Proxly.launch(port) function

**Frontend Structure:**
- Pages folder (NOT features)
- No providers folder - providers from singleton services
- Direct imports for AuthPage and SetupPage (no wrappers)
- Splash page for non-native splash platforms
